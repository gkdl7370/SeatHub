# 8일차 작업 기록

## 오늘의 목표

2주차 첫날에는 바로 기능 구현에 들어가기보다, 이번 주에 구현할 상품, 회차, 좌석의 책임을 먼저 정리했습니다.

1주차에 인증과 공통 구조를 먼저 구성했기 때문에, 2주차부터는 실제 예약 기능의 기반이 되는 도메인을 추가합니다.

## 확인한 1주차 구현 내용

현재까지 구현된 흐름은 다음과 같습니다.

```text
회원가입
-> 로그인
-> JWT Access Token 발급
-> Bearer Token으로 내 정보 조회
```

이 흐름은 앞으로 관리자 API와 예약 API에 인증 정보를 연결하는 기반이 됩니다.

## 오늘 정리한 설계

### 상품

상품은 사용자가 예약하려는 대상입니다.

예를 들어 공연, 클래스, 시설 이용권 같은 단위입니다.

### 회차

회차는 상품이 열리는 시간입니다.

같은 상품이라도 여러 날짜와 시간에 열릴 수 있기 때문에 상품과 분리합니다.

### 좌석

좌석은 특정 회차에서 실제로 예약되는 단위입니다.

같은 좌석 번호라도 회차가 다르면 예약 상태가 달라질 수 있으므로 좌석은 회차에 속하도록 설계합니다.
