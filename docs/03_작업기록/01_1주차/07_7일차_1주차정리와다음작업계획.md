# 7일차 작업 기록

## 1주차 정리

1주차에는 예약 시스템의 기반이 되는 구조를 먼저 만들었습니다.

기능을 빠르게 늘리기보다, 이후 예약과 결제 기능을 안정적으로 붙일 수 있도록 공통 구조와 실행 환경을 먼저 정리했습니다.

## 완료한 작업

### 프로젝트 기준 정리

- 요구사항 초안 작성
- 기술 스택 정리
- 아키텍처 방향 정리
- 패키지 구조 정리
- ERD 초안 작성
- 예약 상태 흐름 정리
- API 명세 초안 작성

### 실행 환경 구성

- Spring Boot 프로젝트 구성
- MySQL, Redis Docker Compose 구성
- local, test 프로파일 분리
- Flyway 기반 DB 마이그레이션 적용
- Actuator health check 설정

### 공통 구조 구성

- 공통 응답 포맷
- 공통 에러 응답 포맷
- 도메인 예외 구조
- GlobalExceptionHandler
- traceId 필터와 요청 추적 구조

### 인증 기능 구성

- 회원가입 API
- 로그인 API
- BCrypt 기반 비밀번호 암호화
- JWT Access Token 발급
- JWT 검증 필터
- 현재 로그인한 회원 정보 조회 API

## 현재 가능한 흐름

현재까지 구현된 API 기준으로 다음 흐름을 실행할 수 있습니다.

```text
회원가입
-> 로그인
-> Access Token 발급
-> Bearer Token으로 내 정보 조회
```

이 흐름은 앞으로 예약, 결제, 관리자 API를 만들 때 기본 인증 흐름으로 사용됩니다.

## 아직 남은 부분

아직 핵심 예약 도메인은 구현 전입니다.

다음 단계부터는 실제 예약 시스템의 중심이 되는 도메인을 추가합니다.

- 상품
- 회차
- 좌석
- 예약
- 결제
- 정산

## 다음 주 작업 방향

2주차에는 상품, 회차, 좌석의 기본 구조를 먼저 만들 계획입니다.

작업 순서는 다음과 같습니다.

```text
상품 등록/조회
-> 회차 등록/조회
-> 좌석 등록/조회
-> 관리자 권한 구조 적용
-> 기본 통합 테스트 추가
```

이 단계가 끝나면 실제 예약 생성 기능을 붙일 수 있는 준비가 됩니다.

## 확인한 내용

1주차 마지막 기준으로 전체 테스트를 실행했습니다.

```text
gradle test
BUILD SUCCESSFUL
```

이후 기능을 추가할 때도 같은 방식으로 작은 단위의 기능을 만들고, 테스트를 통과시킨 뒤 다음 기능으로 넘어갑니다.
